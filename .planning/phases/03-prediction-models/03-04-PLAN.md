    - Test 1: PredictionEngine.predict_game(session, game) stores Prediction rows for game_winner, game_spread, game_total
    - Test 2: PredictionEngine.predict_player_props(session, game, player_id) stores Prediction rows for player_points, player_rebounds, player_assists, player_3pm
    - Test 3: resolve_outcomes(session) finds completed games with unresolved predictions, creates PredictionOutcome rows with actual values
    - Test 4: is_correct for game_winner = 1 if predicted winner matches actual winner, 0 otherwise
    - Test 5: is_correct for player props = 1 if actual within CI, 0 otherwise
    - Test 6: resolve_outcomes skips already-resolved predictions and games not yet "Final"
    - Test 7: resolve_outcomes skips games where home_score or away_score is None
    - Test 8: train_all_models(session, seasons) builds training data from historical games and returns trained model instances
  </behavior>
  <action>
    1. Create `src/hermes/models/training.py`:
       - `build_game_training_data(session, seasons)`: Query completed games (status="Final") for given seasons. For each game, call GamePredictor.build_game_features(). Build features list and targets (spread = home_score - away_score, total = home_score + away_score). Skip games where features are None. Returns dict: {"game_features": [...], "spread_targets": [...], "total_targets": [...]}.
       - `build_player_training_data(session, seasons)`: Query all BoxScores for completed games. For each, call get_features(). Group by stat type. Returns dict: {"points": (features, targets), "rebounds": ..., "assists": ..., "fg3m": ...}. Skip where get_features returns None.
       - `train_all_models(session, seasons=None, models_dir="data/models/")`: Calls both build_ functions, trains GamePredictor, TotalsPredictor, PlayerPropsPredictor. Saves all to models_dir via joblib. Model version = f"v{date.today().isoformat()}". Returns dict of model instances. Creates models_dir if it does not exist.

    2. Create `src/hermes/models/prediction_engine.py`:
       - `PredictionEngine(session, models_dir="data/models/")`: Loads trained models from disk on init. If models not found, raises clear error with instruction to run `hermes predict --train` first.
       - `predict_game(session, game) -> list[Prediction]`: Runs GamePredictor + TotalsPredictor. Creates 3 Prediction rows:
         * game_winner: predicted_value=1 if home win, 0 if away. win_probability from metadata. CI=None for classification.
         * game_spread: predicted_value=spread. CI from model.
         * game_total: predicted_value=total. CI from quantile regression.
         Adds all to session. Returns list.
       - `predict_player_props(session, game, player_id) -> list[Prediction]`: Calls get_features(), runs PlayerPropsPredictor.predict_all(). Creates 4 Prediction rows (player_points/rebounds/assists/3pm). Skips if get_features returns None. Returns list.
       - `predict_today(session) -> dict`: Finds today's games (game_date = today, status != "Final"). For each: predict_game + find roster via BoxScore history + predict_player_props for each player. Returns summary dict with counts. Commits at end.
       - Each Prediction row stores metadata_json = json.dumps(result.metadata).

    3. Create `src/hermes/models/outcome_resolver.py`:
       - `resolve_outcomes(session) -> int`: Query predictions LEFT JOIN prediction_outcomes WHERE outcome is NULL. For each prediction:
         * Check game status == "Final" AND home_score/away_score NOT NULL. Skip otherwise (per Pitfall 7 from research).
         * game_winner: actual = 1 if home_score > away_score else 0. is_correct = 1 if actual == round(predicted_value) else 0.
         * game_spread: actual = home_score - away_score. is_correct = 1 if sign matches.
         * game_total: actual = home_score + away_score. is_correct = 1 if actual within [confidence_lower, confidence_upper].
         * player_*: actual = box_score stat value. is_correct = 1 if actual within CI.
       - Commit and return count of resolved.

    4. Write tests using in-memory DB. Create fixture games and predictions directly (no need for real model training). Mock model loading in PredictionEngine tests.

    - Test 2: Brier score for perfect predictions is 0.0 (game_winner only)
    - Test 3: Brier score for always-wrong predictions approaches 1.0
    - Test 4: compute_metrics for regression types (game_spread, game_total, player_*) returns MAE instead of Brier score
    - Test 5: compute_calibration(session) returns list of (predicted_bucket, actual_rate) tuples for calibration plot data
    - Test 6: compute_metrics with no resolved predictions returns empty/zero metrics gracefully
    - Test 7: Metrics can be filtered by date range

  <action>
    1. Create `src/hermes/models/metrics.py`:
       - `compute_metrics(session, prediction_type=None, start_date=None, end_date=None) -> dict`:
         * Query Prediction + PredictionOutcome with filters.
         * For game_winner: compute hit_rate (fraction is_correct=1), brier_score using sklearn.metrics.brier_score_loss(y_true, y_prob) where y_true=actual outcome, y_prob=win_probability. IMPORTANT: Brier score is ONLY valid for game_winner (binary probabilistic). Do NOT compute Brier for regression types.
         * For regression types (game_spread, game_total, player_*): compute MAE (mean absolute error between predicted_value and actual_value), RMSE, and CI_coverage (fraction of actuals within CI).
         * Returns dict: {prediction_type: {hit_rate, brier_score (or mae/rmse), ci_coverage, total, resolved}}.
         * If prediction_type=None, computes for all types and returns by_type breakdown.
       - `compute_calibration(session, bins=10) -> list[dict]`: Group game_winner predictions by win_probability decile. For each bucket: compute actual win rate. Returns list of {"bin_lower": float, "bin_upper": float, "predicted_avg": float, "actual_rate": float, "count": int}. Uses sklearn.calibration.calibration_curve internally.
       - `format_metrics_report(metrics_dict) -> str`: Pretty-print table for CLI output. Shows type, hit_rate, brier/mae, ci_coverage, count.

    2. Update `src/hermes/cli.py` with new subcommands (follow existing argparse pattern with lazy imports):
       - `hermes predict --train [--seasons 2022-23,2023-24,2024-25]`: Run train_all_models(). Print training summary (samples per model, cross-val scores).
       - `hermes predict --today`: Run PredictionEngine.predict_today(). Print summary (games predicted, players predicted).
       - `hermes predict --resolve`: Run resolve_outcomes(). Print count resolved.
       - `hermes metrics [--type game_winner] [--start-date 2024-01-01] [--end-date 2024-12-31]`: Run compute_metrics() + format_metrics_report(). Print report.
       - All handlers use lazy imports inside the function body.

    3. Write metrics tests with synthetic Prediction + PredictionOutcome rows inserted directly (no real models needed). Test Brier=0 with perfect predictions, Brier~1 with always-wrong, MAE correctness for regression.
