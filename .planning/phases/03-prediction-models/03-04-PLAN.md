  <name>Task 1: Training pipeline, PredictionEngine, and outcome resolver</name>
  <files>
    src/hermes/models/training.py,
    src/hermes/models/prediction_engine.py,
    src/hermes/models/outcome_resolver.py,
    tests/models/test_prediction_engine.py,

  </files>
  <behavior>
    - Test 1: PredictionEngine.predict_game(session, game) stores Prediction rows for game_winner, game_spread, game_total
    - Test 2: PredictionEngine.predict_player_props(session, game, player_id) stores Prediction rows for points, rebounds, assists, 3pm
    - Test 3: resolve_outcomes(session) finds completed games with unresolved predictions, creates PredictionOutcome rows with actual values
    - Test 4: is_correct for game_winner = 1 if predicted winner matches actual winner, 0 otherwise
    - Test 5: is_correct for player props = 1 if actual value is within predicted CI, 0 otherwise (or use over/under logic: 1 if predicted over/under direction matches)
    - Test 6: resolve_outcomes skips already-resolved predictions
    - Test 7: train_all_models(session, seasons) builds training data from historical games and returns trained model instances
  </behavior>
  <action>
    1. Create `src/hermes/models/training.py`:
       - `build_training_data(session, seasons)`: Query completed games, use get_features() + GamePredictor.build_game_features() to build feature/target arrays. Returns dict of {model_name: (features_list, targets_list)}.
       - `train_all_models(session, seasons)`: Calls build_training_data, trains GamePredictor, TotalsPredictor, and 4x PlayerPropPredictor. Saves models to `data/models/` via joblib. Returns dict of trained models.
       - Model versioning: version string = f"v{date.today().isoformat()}" (retrain daily produces new version).

    2. Create `src/hermes/models/prediction_engine.py`:
       - `PredictionEngine(session, models_dir="data/models/")`: Loads trained models from disk.
       - `predict_game(session, game)`: Runs GamePredictor + TotalsPredictor on a Game. Creates 3 Prediction rows (game_winner, game_spread, game_total). Returns list of PredictionResult.
       - `predict_player_props(session, game, player_id)`: Runs PlayerPropPredictor for each stat. Creates 4 Prediction rows. Returns dict of PredictionResult.
       - `predict_today(session)`: Finds today's unplayed games (status != "Final"), runs predict_game + predict_player_props for all players in each game. Returns summary dict.
       - Each Prediction row stores: game_id, prediction_type, player_id (or None for game-level), predicted_value, confidence_lower, confidence_upper, win_probability (game_winner only), model_version, metadata_json (JSON string with extra info).

    3. Create `src/hermes/models/outcome_resolver.py`:
       - `resolve_outcomes(session)`: Query predictions with no PredictionOutcome. For each, check if game is completed (status="Final"). If so:
         * game_winner: actual = 1 if home won, 0 if away won. is_correct = predicted matches actual.
         * game_spread: actual = home_score - away_score. is_correct = sign matches predicted sign.
         * game_total: actual = home_score + away_score. is_correct = actual within CI.
         * player props: actual = box score stat value. is_correct = actual within CI.
       - Returns count of resolved predictions.

    4. Write tests using in-memory DB with fixture games and predictions. Mock model predictions where needed.
  </action>

  </verify>
  <done>PredictionEngine stores all predictions to DB, outcome resolver backfills actuals from completed games, training pipeline builds from historical data</done>
</task>

       - `compute_metrics(session, prediction_type=None, start_date=None, end_date=None)`: Returns dict with:

         * brier_score: mean of (predicted_probability - actual_outcome)^2 for game_winner predictions
         * total_predictions: count of all predictions matching filters
         * total_resolved: count with PredictionOutcome
         * by_type: breakdown per prediction_type if type=None
       - `compute_calibration(session, bins=10)`: Group game_winner predictions by win_probability decile, compute actual win rate per bucket. Returns list of dicts for calibration chart.

       - `hermes predict --resolve`: Run resolve_outcomes(session). Print resolution count.
       - `hermes metrics`: Run compute_metrics(session) and print formatted report.
       - `hermes metrics --type game_winner`: Filter metrics by prediction type.
       - All with lazy imports inside handlers.
