5. Create `src/hermes/dashboard/data_access.py` — the cached data access layer ALL pages use. Functions (all return plain dicts/DataFrames, never ORM objects):

   - `_get_engine()` — `@st.cache_resource`, creates engine from `settings.db_path`
   - `get_todays_games(today_str: str) -> list[dict]` — `@st.cache_data(ttl=300)`, joins games + teams + predictions for given date. Return list of dicts with keys: game_id, game_date, status, home_team_id, away_team_id, home_team, away_team, home_abbr, away_abbr, home_score, away_score. Also include prediction data by pivoting: for each game, collect all prediction rows and add keys win_probability, predicted_spread, predicted_total (from prediction_type = game_winner/game_spread/game_total).
   - `get_player_rolling_stats(player_id: int) -> pd.DataFrame` — `@st.cache_data(ttl=3600)`, returns DataFrame from player_rolling_stats table ordered by game_date.
   - `get_player_recent_games(player_id: int, limit: int = 10) -> list[dict]` — `@st.cache_data(ttl=3600)`, returns recent box scores for a player.
   - `get_player_info(player_id: int) -> dict | None` — `@st.cache_data(ttl=3600)`, returns player name, position, team, etc.
   - `get_all_players() -> list[dict]` — `@st.cache_data(ttl=3600)`, returns all active players (for search/selection).
   - `get_team_standings() -> list[dict]` — `@st.cache_data(ttl=3600)`, query teams with W-L record (count games where team won vs lost), grouped by conference.
   - `get_team_info(team_id: int) -> dict | None` — `@st.cache_data(ttl=3600)`.
   - `get_team_features(team_id: int) -> pd.DataFrame` — `@st.cache_data(ttl=3600)`, returns team_features rows ordered by game_date.
   - `get_team_games(team_id: int, limit: int = 20) -> list[dict]` — `@st.cache_data(ttl=3600)`, recent games for a team.
   - `get_all_teams() -> list[dict]` — `@st.cache_data(ttl=3600)`.
   - `get_predictions_history(prediction_type: str | None, start_date: str | None, end_date: str | None) -> list[dict]` — `@st.cache_data(ttl=300)`, joins predictions + prediction_outcomes + games + teams for tracker page.
   - `get_calibration_data() -> list[dict]` — `@st.cache_data(ttl=3600)`, calls `compute_calibration()` from metrics module (creates its own session).
   - `get_metrics_summary() -> dict` — `@st.cache_data(ttl=3600)`, calls `compute_metrics()` from metrics module.

   KEY RULES (from research):
   - Use `@st.cache_resource` for engine (shared resource)
   - Use `@st.cache_data(ttl=N)` for query results
   - Prefix non-hashable args with `_` to exclude from cache key
   - Return plain dicts/lists/DataFrames, NEVER SQLAlchemy model instances
   - Create session per query in try/finally, always close
   - For `get_calibration_data` and `get_metrics_summary`, create a separate engine+session internally (these call metrics module functions that expect a Session object)
