```python
ROLLING_STATS = ["points", "rebounds", "assists", "steals", "blocks", "turnovers",
                 "fg_pct", "fg3_pct", "ft_pct", "minutes", "plus_minus", "offensive_rebounds"]
WINDOWS = [5, 10, 20]
```

**`compute_rolling_stats(session, player_id, as_of_date=None)`:**
1. Query BoxScore joined with Game, filtered by `BoxScore.player_id == player_id` AND `BoxScore.minutes > 0` (exclude DNPs) AND `BoxScore.minutes IS NOT NULL`. If `as_of_date` is provided, also filter `Game.game_date < as_of_date`. Order by `Game.game_date ASC`.
2. Convert to pandas DataFrame. Compute per-game percentages: `fg_pct = fgm/fga` (using numpy.where for division safety), similarly for `fg3_pct` and `ft_pct`.
3. For each window in [5, 10, 20], compute `.rolling(window, min_periods=1).mean()` for each stat. Also compute `games_available_{window}` as `.rolling(window, min_periods=1).count()` cast to int.
4. CRITICAL temporal rule: The rolling average stored for game N must use only games BEFORE game N. Implementation: after computing rolling values, SHIFT the rolling columns down by 1 row (`df[col] = df[col].shift(1)`). This means game 1 gets NaN (no prior games), game 2 gets game 1's stats, etc. Then fill NaN rolling values for game 1 with NULL (first game has no prior data).
5. Build PlayerRollingStats objects from the DataFrame. Upsert using `session.merge()` keyed on (player_id, game_id).
6. Commit after all rows for a player are upserted.

**`compute_rolling_stats_for_games(session, game_ids)`:**
Convenience: find all player_ids in those games' box scores, call compute_rolling_stats for each player.

DNP handling per research: skip games where minutes is 0 or NULL. This is the NBA analytics consensus.
Trade handling per research: carry stats across trades (do not reset). The rolling window naturally decays pre-trade games.

Update test stubs in `tests/data/test_rolling_features.py` -- replace `pytest.skip()` with real test implementations using the fixtures from conftest.py. Each test creates specific box scores with known values and asserts exact rolling average outputs.

**DB-integrated function:**
- `compute_advanced_stats(session, player_id, as_of_date=None)`:
  1. Query BoxScore for player, joined with Game, filter `minutes > 0`. If as_of_date provided, filter `game_date < as_of_date`.

The as_of_date filter ensures we don't compute stats for games that haven't happened yet, but each game's stats are computed from its own box score. The temporal concern here is about team aggregates -- when computing team totals for a game, use only box scores from THAT game (not future games). This is naturally correct since we filter by game_id.
