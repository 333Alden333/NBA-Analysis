1. Create `src/hermes/agent/tui.py` with `run_chat(session, model_override=None)`:
   - Import prompt_toolkit.PromptSession, prompt_toolkit.history.FileHistory, rich.console.Console, rich.markdown.Markdown
   - On entry: check Ollama connection via `check_ollama_connection()`. If unreachable, print rich-formatted error with install instructions ("Install Ollama: curl -fsSL https://ollama.com/install.sh | sh" and "Pull model: ollama pull llama3.1:8b") and exit.
   - Create agent via `create_agent(session)` (pass model_override if provided)
   - Create PromptSession with FileHistory(".hermes_history")
   - Create Rich Console
   - Print welcome banner: "Hermes Agent - NBA Analytics Assistant" with brief help text listing example queries
   - Main loop:
     a. Prompt "You> " using session.prompt()
     b. Handle exit commands: "exit", "quit", "/q" (case-insensitive, stripped)
     c. Handle empty input (skip, re-prompt)
     d. Handle "/help" command -- print available commands and example queries
     e. Call `agent.run(user_input, reset=False)` for multi-turn conversation continuity
     f. Catch exceptions from agent.run() gracefully -- print error message, continue loop (don't crash)
     g. Format response with Rich: use Console.print() with cyan color for "Hermes>" prefix. If response contains markdown-like content (headers, bullets, tables), render as Markdown.
     h. Catch KeyboardInterrupt and EOFError to exit cleanly with goodbye message

2. Update `src/hermes/cli.py`:
   - Add "chat" subcommand to build_parser():
     ```python
     chat_parser = sub.add_parser("chat", help="Interactive Hermes agent chat")
     chat_parser.add_argument(
         "--model", type=str, default=None,
         help="Ollama model to use (default: from config or llama3.1:8b)",
     )
     ```
   - Update the command validation in main() to include "chat" in the allowed commands
   - Add handler block for args.command == "chat":
     ```python
     if args.command == "chat":
         from hermes.agent.tui import run_chat
         session = _create_session()
         try:
             run_chat(session, model_override=args.model)
         finally:
             session.close()
         return
     ```
   - Place the chat handler BEFORE the session creation block for non-interactive commands (after the dashboard handler, before `session = _create_session()` -- actually chat needs a session, so place it in the try block similar to other commands)
