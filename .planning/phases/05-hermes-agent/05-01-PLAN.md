1. Create `src/hermes/agent/tools.py` with 8 smolagents Tool subclasses. Each tool:
   - Inherits from `smolagents.Tool`
   - Has `name`, `description` (detailed with examples), `inputs` dict, `output_type = "string"`
   - Accepts `db_session` in `__init__`, stores as `self._session`
   - Calls `super().__init__()` in __init__
   - `forward()` method calls data_queries function, formats result, returns string
   - Handles empty/None results gracefully

   Tools to create:
   a. `SearchPlayer` -- fuzzy name search. Input: player_name (string). Uses search_players().
   b. `GetPlayerStats` -- recent games for a player. Input: player_name (string). Resolves name via search first, gets top match, then gets recent games + formats.
   c. `GetPlayerPredictions` -- predictions for a player. Input: player_name (string). Same name resolution pattern.
   d. `GetTeamInfo` -- team standings and info. Input: team_name (string). Fuzzy match against team full_name and abbreviation.
   e. `GetTodayGames` -- today's game slate. No inputs (uses date.today()).
   f. `GetPredictionAccuracy` -- model accuracy metrics. Input: prediction_type (string, nullable).
   g. `GetPredictionHistory` -- past predictions with outcomes. Input: prediction_type (string, nullable).
   h. `GetMatchupAnalysis` -- player vs team history. Inputs: player_name (string), team_name (string). Resolves both, queries matchup data.

   IMPORTANT: For name resolution in tools, use difflib.get_close_matches on full_name list. If no close match, return "Player/Team not found. Did you mean: {closest alternatives}?" message.
